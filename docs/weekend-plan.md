# Weekend Build Plan

## Saturday morning: repository and local app

- Create GitHub repository.
- Install dependencies.
- Run Next.js locally.
- Confirm the page renders.

## Saturday afternoon: Supabase

- Create Supabase project.
- Apply `db/schema.sql`.
- Set `.env.local`.
- Save the first choice from localhost.

## Sunday morning: Cloudflare Pages

- Push to GitHub.
- Connect Cloudflare Pages.
- Set environment variables.
- Deploy.

## Sunday afternoon: first reflection

Write a short note answering:

- Did the stack feel light enough?
- Was Supabase understandable?
- Did Cloudflare Pages feel easier than self-hosting?
- What feature is tempting but should still be postponed?

## Definition of done

The service is done for the weekend if:

- It accepts one choice.
- It stores the choice.
- It shows the latest choices.
- It is deployed through Cloudflare Pages.
